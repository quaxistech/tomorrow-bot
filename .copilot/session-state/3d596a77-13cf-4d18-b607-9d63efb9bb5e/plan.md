# Tomorrow Bot v0.3 — «Бот будущего»

## Проблема
Текущий бот имеет профессиональную архитектуру (36 модулей, 15 проверок риска, 5 стратегий),
но тактически неполон: фиксированные стопы теряют профит, статический сайзинг не учитывает
волатильность, стратегии не адаптируются к деградации, нет передовых техник анализа рынка.

## Подход
Три уровня улучшений — от критических (прямой $$-эффект) до передовых (опережение рынка).

---

## УРОВЕНЬ 1 — Критические улучшения (прямая прибыль)

### 1.1. Адаптивный стоп-лосс: Chandelier Exit + Breakeven + Partial Take-Profit
**Текущее**: фиксированный 2×ATR стоп → прибыльные сделки закрываются слишком рано
**Новое**:
- [ ] **Trailing Stop (Chandelier Exit)**: `stop = highest_since_entry - N×ATR(14)`
  - В тренде (ADX>25): N=3.0 (широкий, даём расти)
  - В боковике (ADX<20): N=1.5 (узкий, быстрый выход)
  - Стоп ТОЛЬКО подтягивается вверх, никогда не опускается
- [ ] **Breakeven Stop**: при достижении +1.5×ATR профита → стоп переносится на entry+0.1×ATR
- [ ] **Partial Take-Profit**: при +2×ATR → закрытие 50% позиции, остаток с трейлингом
- [ ] **Regime-Aware множитель**: ATR mult зависит от режима рынка

### 1.2. Динамический сайзинг: Volatility Targeting
**Текущее**: фиксированный % капитала → одинаковый риск в штиль и шторм
**Новое**:
- [ ] **Volatility Target**: `size = capital × (target_vol / realized_vol_20) × base_risk_pct`
  - target_vol = 2% (daily, конфигурируемо)
  - Если рынок спокойный (vol=1%) → удвоенная позиция
  - Если рынок бешеный (vol=10%) → уменьшенная позиция в 5 раз
- [ ] **Kelly Criterion (fractional)**: `f* = win_rate - (1-win_rate)/payoff_ratio`, используем 25% Kelly
- [ ] **Regime-Based Sizing**: ChopNoise → 50% размера, StableTrend → 100%, LiquidityVacuum → 0%

### 1.3. Alpha Decay → Auto-Tuning Feedback Loop
**Текущее**: alpha decay мониторит, но НЕ действует
**Новое**:
- [ ] **Auto-Reduce**: если Z-score drift > 2σ → размер позиции -50%
- [ ] **Auto-Disable**: если Z-score > 3σ → стратегия отключается на 4 часа
- [ ] **Auto-Tighten**: при деградации → conviction threshold +0.1
- [ ] **Feedback в RiskEngine**: degradation_factor (0.0-1.0) влияет на max_notional

### 1.4. HTF Real-Time Update
**Текущее**: HTF загружается ТОЛЬКО при старте (один раз)
**Новое**:
- [ ] Пересчёт HTF тренда каждые 1 час (при закрытии часовой свечи)
- [ ] Отслеживание candle1h через WebSocket
- [ ] При резком изменении тренда → экстренное обновление (не ждать час)

## УРОВЕНЬ 2 — Продвинутые техники (конкурентное преимущество)

### 2.1. CUSUM Regime Change Detection
**Текущее**: жёсткие правила, резкие переходы
**Новое**:
- [ ] **CUSUM (Cumulative Sum)**: обнаруживает структурные сдвиги на 1-2 бара раньше
  - Следит за накоплением отклонений от среднего
  - При пробитии порога → сигнал смены режима
- [ ] **Интеграция с RegimeEngine**: CUSUM даёт early warning, RegimeEngine подтверждает

### 2.2. VPIN — Volume-Synchronized Probability of Informed Trading
**Текущее**: простой buy/sell ratio
**Новое**:
- [ ] **VPIN**: профессиональный индикатор токсичности потока ордеров
  - Измеряет дисбаланс информированных vs шумовых трейдеров
  - VPIN > 0.7 → высокий риск adverse selection → блокировка входа
  - Используется хедж-фондами (Easley, López de Prado, O'Hara, 2012)
- [ ] **Интеграция**: в FeatureSnapshot как `vpin_score`, в стратегии как фильтр

### 2.3. Volume Profile Analysis
**Текущее**: нет анализа объёмных уровней
**Новое**:
- [ ] **Point of Control (POC)**: цена с максимальным объёмом за последние 24ч
- [ ] **Value Area (VA)**: диапазон 70% объёма — зона «справедливой цены»
- [ ] **Naked POC**: непротестированные POC от предыдущих сессий → уровни поддержки/сопротивления
- [ ] **Использование**: entry score повышается при входе рядом с POC/VA edge

### 2.4. Smart Execution — Adaptive TWAP
**Текущее**: один ордер market/limit
**Новое**:
- [ ] **Mini-TWAP**: разбиение ордера на 3-5 частей через 2-5 секунд
- [ ] **Adaptive Urgency**: если цена убегает → ускорить (aggressive), стоит → замедлить (passive)
- [ ] **Fill Probability Model**: оценка вероятности исполнения limit-ордера за T секунд
  - Если P(fill) < 30% → переключить на market

### 2.5. Time-of-Day Alpha
**Текущее**: торгует 24/7 одинаково
**Новое**:
- [ ] **Сессионный профиль**: Asian (00-08 UTC), European (08-16), American (16-00)
  - Asian: низкая волатильность → скальпинг предпочтителен
  - US Open (13:30-16:00 UTC): максимальная волатильность → momentum
  - Overlap (12-16 UTC): максимальная ликвидность → лучшее исполнение
- [ ] **Session-Adjusted Sizing**: в тихие часы → меньше размер, меньше ATR mult

## УРОВЕНЬ 3 — «Бот будущего» (опережение рынка)

### 3.1. Bayesian Online Parameter Adaptation
**Что это**: вместо фиксированных порогов (RSI<30 = buy), система непрерывно
обновляет оптимальные пороги на основе байесовского вывода.
**Зачем**: рынок меняется — то, что работало вчера, может не работать сегодня.
- [ ] **Adaptive RSI thresholds**: начинаем с prior (30/70), обновляем по результатам сделок
- [ ] **Adaptive EMA periods**: если 20/50 не работает → байесовский сдвиг к 15/40 или 25/60
- [ ] **Rolling posterior**: последние 100 сделок формируют belief distribution
- [ ] **Exploration/Exploitation**: 90% — лучшие параметры, 10% — исследование альтернатив

### 3.2. Information-Theoretic Signal Quality (Entropy Filter)
**Что это**: энтропия Шеннона применяется к потоку сигналов для оценки «информативности».
**Зачем**: когда рынок шумит — все индикаторы дают случайные сигналы. Энтропийный фильтр это ЛОВИТ.
- [ ] **Signal Entropy**: H = -Σ p(x) log p(x) по последним N решениям стратегии
  - Высокая энтропия → сигналы случайны → не торговать
  - Низкая энтропия → сигналы согласованы → торговать уверенно
- [ ] **Cross-Strategy Information**: взаимная информация I(X;Y) между стратегиями
  - Если стратегии дают независимые одинаковые сигналы → conviction boost
  - Если коррелированы → не усиливать (избыточная информация)

### 3.3. Microstructure Regime Detection (Order Flow Fingerprinting)
**Что это**: идентификация «отпечатков» различных участников рынка по паттернам ордеров.
**Зачем**: маркет-мейкеры, арбитражёры, ритейл — у каждого свой «отпечаток».
- [ ] **Spoofing Detection**: обнаружение фейковых ордеров (появляются и исчезают за <1с)
- [ ] **Iceberg Detection**: когда крупный ордер скрыт за мелкими — отслеживать частоту refill
- [ ] **Aggressive Flow Clustering**: кластеризация агрессивных сделок по размеру и частоте
- [ ] **Использование**: токсичный flow → расширение стопа или выход из позиции

### 3.4. Liquidation Cascade Prediction
**Что это**: прогнозирование каскадных ликвидаций на фьючерсном рынке по давлению на спот.
**Зачем**: ликвидационные каскады создают экстремальные движения — можно как избежать их, так и заработать.
- [ ] **Open Interest Proxy**: отслеживание изменения объёмов как proxy для позиционирования
- [ ] **Cascade Risk Score**: [0-1], если > 0.7 → закрыть все позиции или расширить стоп до 5×ATR
- [ ] **Contra-Cascade Entry**: после каскада (определяется по spike volume + RSI < 10) → вход на отскок

### 3.5. Multi-Asset Correlation Monitor
**Что это**: отслеживание корреляций между топ-парами в реальном времени.
**Зачем**: когда корреляции рушатся — это ГЛАВНЫЙ сигнал смены рыночного режима.
- [ ] **Rolling Correlation Matrix**: 1ч корреляции между top-5 парами
- [ ] **Correlation Breakdown Alert**: если корреляция BTC/ETH падает с 0.9 до 0.3 → смена режима
- [ ] **Decorrelation Entry**: при восстановлении корреляции после разрыва → сильный momentum signal

### 3.6. Reinforcement Learning для тайминга входа
**Что это**: RL-агент учится выбирать оптимальный момент входа на основе состояния рынка.
**Зачем**: вместо жёстких правил (RSI<30) → система учится из опыта.
- [ ] **State**: [RSI, MACD, vol, regime, HTF_trend, VPIN, time_of_day]
- [ ] **Actions**: [enter_now, wait_1min, wait_5min, skip]
- [ ] **Reward**: realized P&L сделки
- [ ] **Algorithm**: Contextual Bandit (Thompson Sampling) — лёгкий, не требует GPU
- [ ] **Safety**: RL только ОТКЛАДЫВАЕТ вход, никогда не СОЗДАЁТ сигналы (стратегии решают что, RL решает когда)
