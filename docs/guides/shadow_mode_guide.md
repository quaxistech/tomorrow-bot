# Руководство по теневому режиму (Shadow Mode)

## Обзор

ShadowModeEngine позволяет стратегиям и вариантам решений работать виртуально,
без размещения реальных ордеров. Теневые решения записываются с полным контекстом
для последующего сравнения с live-результатами.

## Гарантия безопасности

**КРИТИЧЕСКИ ВАЖНО**: теневой режим НИКОГДА не размещает реальные ордера.
Это обеспечивается архитектурно:
- ShadowModeEngine не имеет доступа к IOrderSubmitter
- Все решения записываются как `ShadowDecision` (read-only)
- Нет пути исполнения из теневого контура в реальный

## Жизненный цикл теневого решения

```
1. Стратегия генерирует сигнал
2. record_shadow_decision() → ShadowTradeRecord создан
3. update_price_tracking() вызывается периодически
   → Цена через 1с, 5с, 30с записывается
   → Гипотетический P&L рассчитывается
4. compare() → ShadowComparison с live-результатами
```

## Отслеживание цены

После записи теневого решения система отслеживает реальную цену:

| Временная метка | Поле | Назначение |
|-----------------|------|-----------|
| t=0 | market_price_at_decision | Цена в момент решения |
| t+1с | market_price_after_1s | Краткосрочное движение |
| t+5с | market_price_after_5s | Среднесрочное движение |
| t+30с | market_price_after_30s | P&L proxy |

Гипотетический P&L (bps) рассчитывается по формуле:
```
pnl_bps = (price_30s - intended_price) / intended_price × 10000
```
С инверсией знака для Sell-решений.

## Сравнение Shadow vs Live

```cpp
ShadowComparison {
    shadow_trades,     // Количество теневых сделок
    live_trades,       // Количество реальных сделок
    shadow_pnl_bps,    // Совокупный P&L теневых
    live_pnl_bps,      // Совокупный P&L live
    shadow_hit_rate,   // % прибыльных теневых
    live_hit_rate      // % прибыльных live
};
```

## Конфигурация

```cpp
ShadowConfig {
    enabled = false,           // Выключен по умолчанию
    max_shadow_records = 10000 // Лимит записей на стратегию
};
```

При достижении лимита старейшие записи вытесняются (FIFO).

## Применение

- **Тестирование новых стратегий** — запуск в shadow до доверия
- **A/B тестирование** — параллельное сравнение вариантов
- **Режим наблюдения** — система работает, но не торгует
- **Исследования** — анализ «что было бы, если»
