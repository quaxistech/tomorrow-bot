# Руководство по Operator Control

## Обзор

Модуль `OperatorControlPanel` предоставляет оператору интерфейс управления
системой Tomorrow Bot с полным аудитом каждого действия.

---

## 1. Список команд оператора

### Управление торговлей

| Команда                  | Описание                                    |
|--------------------------|---------------------------------------------|
| `kill_switch on/off`     | Аварийная остановка всей торговли            |
| `safe_mode on/off`       | Режим «только чтение» (мониторинг без торг.) |
| `set_trading_mode <m>`   | Изменить режим: paper, shadow, production    |

### Управление стратегиями

| Команда                       | Описание                         |
|-------------------------------|----------------------------------|
| `enable_strategy <id>`        | Включить стратегию               |
| `disable_strategy <id> <why>` | Выключить стратегию с причиной   |
| `register_strategy <id> <v>`  | Зарегистрировать новую стратегию |

### Конфигурация

| Команда                     | Описание                            |
|-----------------------------|-------------------------------------|
| `update_config_hash <hash>` | Обновить хеш конфигурации           |
| `set_runtime_version <ver>` | Установить версию runtime           |

---

## 2. Безопасность

### Аудит каждого действия

Все команды автоматически записываются в `GovernanceAuditLayer`:

```
[2024-01-15 10:30:00.123] AUDIT: KillSwitchActivated
  actor=operator target=system
  details="Активация kill switch: подозрительная активность"
```

### Принцип наименьших привилегий

- Kill switch может активировать любой оператор
- Деактивация kill switch требует явного подтверждения
- Изменение trading mode на `production` логируется с повышенным приоритетом

### Защита от ошибок

- Двойная активация kill switch — идемпотентна
- Включение уже включённой стратегии — без ошибки
- Выключение несуществующей стратегии — ошибка с логированием

## 3. Inspect-команды

### Состояние системы

| Команда                  | Описание                          |
|--------------------------|-----------------------------------|
| `get_snapshot`           | Полный снимок governance          |
| `get_strategy_info <id>` | Информация о стратегии            |
| `get_all_strategies`     | Список всех стратегий             |
| `get_audit_log <n>`     | Последние N записей аудита        |

### Health и метрики

| Команда           | Описание                         |
|-------------------|----------------------------------|
| `health`          | Общее состояние системы          |
| `health_detail`   | Состояние каждой подсистемы      |
| `metrics`         | Текущие метрики (Prometheus)     |

## 4. Примеры использования

### Плановое отключение стратегии

```
1. operator> disable_strategy momentum_v1 "Плановое обслуживание"
2. operator> get_strategy_info momentum_v1
   → enabled=false, reason="Плановое обслуживание"
3. ... (выполнить обслуживание) ...
4. operator> enable_strategy momentum_v1
5. operator> get_audit_log 5
   → Видим записи disable + enable
```

### Экстренная остановка

```
1. operator> kill_switch on "Аномальная активность рынка"
2. → Все ордера отменены, новые блокированы
3. operator> get_snapshot
   → kill_switch_active=true
4. ... (дождаться нормализации) ...
5. operator> kill_switch off
```

---

## 5. Интеграция

- Operator Control вызывает Governance для всех действий
- Governance записывает аудит и обновляет состояние
- Supervisor учитывает kill switch и safe mode
- Risk Guard проверяет governance флаги перед ордерами
