# Архитектура персистентности

## Обзор

Слой персистентности Tomorrow Bot обеспечивает надёжное сохранение всех торговых событий,
снимков состояний, решений и телеметрии для последующего анализа, воспроизведения и аудита.

## Компоненты

### IStorageAdapter — абстракция хранилища
Интерфейс, определяющий контракт взаимодействия с хранилищем:
- `append_journal()` — добавить запись в журнал событий
- `query_journal()` — запросить записи по временному диапазону и типу
- `store_snapshot()` — сохранить снимок состояния
- `load_latest_snapshot()` — загрузить последний снимок указанного типа
- `flush()` — принудительный сброс буферов

### MemoryStorageAdapter — реализация в памяти
Потокобезопасная (std::mutex) реализация для тестирования и начального развёртывания.
Хранит данные в `std::vector<JournalEntry>` и `std::unordered_map` для снимков.

### EventJournal — журнал событий
Append-only журнал с автоматической генерацией монотонных sequence_id (std::atomic).
- Каждая запись содержит: тип, метку времени, correlation_id, strategy_id, config_hash, JSON payload
- Поддерживает 8 типов записей: MarketEvent, DecisionTrace, RiskDecision, OrderEvent,
  PortfolioChange, StrategySignal, SystemEvent, TelemetrySnapshot

### SnapshotStore — хранилище снимков
Сохраняет и загружает снимки состояний с автоматической нумерацией.
- 5 типов снимков: Portfolio, RiskCounters, StrategyMeta, WorldState, FullSystem

### PersistenceLayer — фасад
Объединяет EventJournal + SnapshotStore в единый интерфейс:
- `journal()` — доступ к журналу событий
- `snapshots()` — доступ к снимкам
- `flush()` — сброс всех буферов
- `is_enabled()` — проверка конфигурации

## Границы персистентности

| Данные | Тип хранения | Частота |
|--------|-------------|---------|
| Активные ордера | Журнал (OrderEvent) | При каждом изменении |
| Позиции | Снимок (Portfolio) | Периодически |
| Счётчики риска | Снимок (RiskCounters) | При изменении |
| Метаданные стратегий | Снимок (StrategyMeta) | При обновлении |
| Версии стратегий | Журнал (StrategySignal) | При регистрации |
| Хэш конфигурации | В каждой записи | Всегда |
| Трассировки решений | Журнал (DecisionTrace) | При каждом решении |
| Журнал событий | Журнал (все типы) | Непрерывно |
| Снимки телеметрии | Журнал (TelemetrySnapshot) | Периодически |

## Модульность хранилища

Система спроектирована для лёгкой замены хранилища. Для перехода на файловое или
сетевое хранилище достаточно реализовать IStorageAdapter:
- `FileStorageAdapter` — JSON Lines файлы (будущее расширение)
- `PostgresStorageAdapter` — PostgreSQL (будущее расширение)
- `S3StorageAdapter` — AWS S3 (будущее расширение)

## Потокобезопасность

Все публичные методы защищены std::mutex. Безопасно для многопоточного доступа.
