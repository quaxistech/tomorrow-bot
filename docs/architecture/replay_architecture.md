# Архитектура движка воспроизведения (ReplayEngine)

## Обзор

ReplayEngine обеспечивает детерминированное воспроизведение исторических событий
из журнала персистентности для анализа, отладки и верификации решений.

## Сценарии использования

1. **Детерминированный анализ** — воспроизведение захваченных рыночных данных
2. **Реконструкция решений** — восстановление контекста принятия решений
3. **Post-mortem анализ** — разбор проблемных ситуаций
4. **Анализ challenger'ов** — сравнение альтернативных стратегий
5. **Регрессионная верификация** — проверка воспроизводимости результатов

## Состояния движка

```
Idle → Playing → Completed
  ↑       ↓
  └── Paused
         ↓
       Error
```

- **Idle** — начальное состояние, ожидание конфигурации
- **Playing** — воспроизведение событий
- **Paused** — пауза (зарезервировано для будущего)
- **Completed** — все события воспроизведены
- **Error** — ошибка при воспроизведении

## Конфигурация воспроизведения

```cpp
ReplayConfig {
    start_time,              // Начало временного диапазона
    end_time,                // Конец временного диапазона
    speed_factor,            // 1.0 = реальное, 0 = максимальная скорость
    strategy_filter,         // Опциональный фильтр по стратегии
    reconstruct_decisions,   // Реконструировать контекст решений
    emit_telemetry           // Генерировать телеметрию при воспроизведении
};
```

## Интерфейс

- `configure(ReplayConfig)` — настройка параметров воспроизведения
- `start()` — загрузка событий из хранилища, начало воспроизведения
- `step()` — получить следующее событие (пошаговый режим)
- `has_next()` — проверка наличия следующего события
- `get_result()` — итоговый отчёт воспроизведения
- `get_state()` — текущее состояние движка
- `reset()` — сброс в начальное состояние

## Реконструкция путей

Воспроизведение может реконструировать (в рамках практических ограничений):
- Нормализованные рыночные события
- Путь генерации фич (FeatureSnapshot)
- Выходы WorldModel и Regime
- Предложения стратегий
- Результаты агрегации решений
- Решения риск-менеджера
- Решения по исполнению
- Трассировки телеметрии

## Ограничения текущей реализации

- Воспроизведение из in-memory хранилища (MemoryStorageAdapter)
- Нет поддержки ускорения/замедления в реальном времени (speed_factor зарезервирован)
- Реконструкция решений помечает записи типа DecisionTrace как реконструированные
- Файловое хранилище — будущее расширение
