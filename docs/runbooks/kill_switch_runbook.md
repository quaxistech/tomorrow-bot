# Kill Switch — руководство оператора

## Что такое Kill Switch

Kill Switch — аварийный механизм немедленной остановки всех торговых операций.
При активации система немедленно отменяет все открытые ордера и прекращает
генерацию новых.

## Когда активировать Kill Switch

- Резкое движение рынка более 10% за 5 минут (flash crash)
- Обнаружение аномального поведения системы
- Потеря связи с биржей более 60 секунд
- Превышение дневного лимита убытков
- Любая техническая неисправность
- По требованию регулятора

## Способы активации

### 1. Автоматический (риск-менеджер)
Система автоматически активирует Kill Switch при:
- Превышении `max_daily_loss_pct`
- Превышении `max_drawdown_pct`
- Обнаружении аномалий рынка (Phase 6)

### 2. Сигнал завершения
```bash
# Graceful shutdown — ждёт завершения текущих операций
kill -SIGTERM $(pidof tomorrow-bot)

# Немедленное завершение
kill -SIGINT $(pidof tomorrow-bot)
```

### 3. Через systemd
```bash
sudo systemctl stop tomorrow-bot
```

## Проверка состояния после активации

```bash
# Проверить статус системы
curl http://localhost:8080/health

# Проверить лог
journalctl -u tomorrow-bot --since "5 minutes ago"

# Проверить метрики
curl http://localhost:9090/metrics | grep kill_switch
```

## Действия после активации

1. Убедитесь что все ордера отменены (проверьте в интерфейсе биржи)
2. Сохраните логи: `journalctl -u tomorrow-bot > incident_$(date +%Y%m%d_%H%M).log`
3. Определите причину активации
4. Зафиксируйте инцидент в системе отслеживания

## Возобновление работы

1. Устраните причину активации Kill Switch
2. Проверьте конфигурацию
3. Запустите систему в paper режиме для верификации
4. При успешной проверке — переключите в production
