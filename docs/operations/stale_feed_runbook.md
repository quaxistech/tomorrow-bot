# Runbook: Устаревшие рыночные данные (Stale Feed)

## Обзор

Описывает действия при обнаружении устаревших рыночных данных от биржи.

---

## 1. Симптомы

- Метрика `tb_feed_staleness_ms` превышает порог (по умолчанию 5000 мс)
- Лог: `"Рыночные данные устарели"` (уровень warn)
- Health подсистемы `market_data` переходит в состояние `Degraded`
- Спред может расти из-за неактуальных цен
- Индикаторы перестают обновляться

---

## 2. Автоматическая реакция

### Risk Guard

- Блокировка новых ордеров при `staleness > 3000ms`
- Снижение уверенности стратегий на 50%
- Повышение порога входа в 2 раза

### Adversarial Defense

- `StaleFeedBurst` детектор активируется
- При серии устаревших обновлений — `VetoTrade`
- Cooldown на 30 секунд после восстановления

### Supervisor

- Автоматический переход в деградированный режим
- Уведомление оператора через логи и алерты

---

## 3. Ручные действия

### Диагностика

```bash
# Проверить подключение к бирже
curl http://localhost:9090/health/subsystems | jq '.market_data'

# Проверить метрику устаревания
curl -s http://localhost:9090/metrics | grep tb_feed_staleness

# Проверить логи WebSocket
journalctl -u tomorrow-bot | grep -i "websocket\|stale\|feed"
```

### Действия оператора

1. Проверить статус биржи (status.bitget.com)
2. Если биржа работает — перезапустить WebSocket подключение
3. Если биржа недоступна — активировать safe mode
4. При длительной недоступности — активировать kill switch

---

## 4. Восстановление

### Автоматическое

- При получении свежих данных система автоматически выходит из деградации
- Cooldown 30 секунд перед возобновлением торговли
- Проверка согласованности стакана

### Ручное

```
operator_control: safe_mode off
```

### Проверка после восстановления

- [ ] Health подсистемы `market_data` → Healthy
- [ ] Метрика staleness < 1000 мс
- [ ] Спред в нормальном диапазоне
- [ ] Стакан валиден и синхронизирован
- [ ] Стратегии возобновили генерацию сигналов

---

## 5. Предотвращение

- Мониторинг задержки WebSocket соединения
- Резервный источник данных (scaffolded)
- Алерты при staleness > 2000 мс (предупреждение)
